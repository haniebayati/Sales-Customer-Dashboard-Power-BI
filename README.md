# Power-BI-Sales-Customer-Dashboard(پروژه تمرینی شخصی)



## ساختار پروژه

### 1. نمای کلی داشبورد (عملکرد فروش)
 **هدف:** رصد شاخص‌های فروش و روند سود در مناطق و دسته‌بندی‌های مختلف محصول.

 **ویژگی‌ها:**
  
   **KPIها:** مجموع فروش، تعداد سفارشات، حاشیه سود، مجموع حاشیه
 
   **ویژوال‌ها:**  
     روند فروش و سود (نمودار خطی و ستونی ترکیبی)  
     نمودار دایره‌ای حاشیه سود بر اساس دسته‌بندی محصول
     
   **فیلترها (Slicer):** منطقه، دسته‌بندی محصول، سال
   
   **تعامل:** بروزرسانی دینامیک تمام ویژوال‌ها و KPIها با انتخاب فیلترها

### 2. تحلیل مشتریان و سفارشات
 **هدف:** تحلیل رفتار، جمعیت‌شناسی و الگوهای سفارش مشتریان.
 
 **ویژگی‌ها:**

   **KPIها:** تعداد کل مشتریان، مشتریان جدید، میانگین ارزش سفارش، سفارشات به ازای هر مشتری
  
   **ویژوال‌ها:**  
     جمعیت‌شناسی مشتریان (نمودار ستونی بر اساس سن و جنسیت)  
     روند سفارشات (نمودار خطی + ستونی ماهانه)  
     نمای کلی کانال فروش (آنلاین، حضوری، نماینده فروش)  
     رضایت و امتیاز مشتریان (گیج‌ها برای میانگین امتیاز و رضایت)
     
   **تعامل:** کلیک روی هر کانال یا گروه سنی، دیگر ویژوال‌ها را بروزرسانی می‌کند

### 3. نمای مدل داده
 **هدف:** مدلسازی داده و ایجاد روابط بین جداول.

 **جداول:**
  - **Customers:** شناسه مشتری، نام، سن، جنسیت، مکان  
  - **Orders:** شناسه سفارش، شناسه مشتری، شناسه محصول، تاریخ سفارش، تعداد، فروش، سود  
  - **Products:** شناسه محصول، نام محصول، دسته‌بندی، قیمت  
  - **Region:** شناسه منطقه، نام منطقه  
  - **Date:** تاریخ، سال، ماه، روز

 **روابط:**  
  - Customers → Orders (یک به چند)  
  - Products → Orders (یک به چند)  
  - Region → Orders (یک به چند)  
  - Date → Orders (یک به چند)

**اصول طراحی:** استفاده از **Star Schema**، کاهش افزونگی و بهینه‌سازی محاسبات DAX
